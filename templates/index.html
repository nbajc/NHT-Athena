2. AUTHENTICATION

Method: Email + password, session-based
No OAuth, no third-party SSO — sovereign by design

Implementation

POST /auth/login
  body: { email, password }
  → validates against USERS table in Supabase
  → issues signed JWT stored as httpOnly cookie (7-day expiry)
  → returns { user_id, display_name }

POST /auth/logout
  → clears session cookie

GET /auth/verify
  → checks cookie validity on every protected route

Supabase: USERS table

CREATE TABLE users (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  email TEXT UNIQUE NOT NULL,
  password_hash TEXT NOT NULL,  -- bcrypt
  display_name TEXT,
  created_at TIMESTAMPTZ DEFAULT NOW(),
  last_login TIMESTAMPTZ
);

3. SUPABASE MEMORY ARCHITECTURE

Athena remembers everything. Memory is not a session — it is a persistent, queryable, semantic record of the operator's context, decisions, and state.

Tables

-- Conversation history: every message, forever
CREATE TABLE messages (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id UUID REFERENCES users(id),
  role TEXT CHECK (role IN ('user','assistant','system')),
  content TEXT NOT NULL,
  model TEXT,               -- 'claude' | 'gemini'
  panel TEXT,               -- 'core' | 'brief' | 'comms' | 'today'
  embedding VECTOR(1536),   -- pgvector semantic search
  created_at TIMESTAMPTZ DEFAULT NOW()
);

-- Long-horizon goals
CREATE TABLE horizon_items (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id UUID REFERENCES users(id),
  target_date DATE,
  title TEXT NOT NULL,
  description TEXT,
  urgency TEXT CHECK (urgency IN ('critical','high','medium','low')),
  status TEXT DEFAULT 'active',
  created_at TIMESTAMPTZ DEFAULT NOW(),
  updated_at TIMESTAMPTZ DEFAULT NOW()
);

-- Brief items (active strategic tasks)
CREATE TABLE brief_items (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id UUID REFERENCES users(id),
  content TEXT NOT NULL,
  urgency TEXT,
  status TEXT DEFAULT 'active',
  sort_order INT,
  created_at TIMESTAMPTZ DEFAULT NOW()

-- Today's timeline blocks

  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),

  block_date DATE DEFAULT CURRENT_DATE,
  time_slot TEXT,
  title TEXT NOT NULL,
  notes TEXT,
  completed BOOLEAN DEFAULT FALSE,
  created_at TIMESTAMPTZ DEFAULT NOW()

-- Side tasks: autonomous backend agents

  next_run TIMESTAMPTZ,

4. INTELLIGENCE ORCHESTRATION

Gemini and Claude are not interchangeable. They have distinct roles. The backend routes work to their respective strengths.

Model Assignment

Task Model Reason
Daily brief synthesis Gemini 2.0 Flash Speed, breadth, pattern aggregation
Email parsing + classification Gemini 2.0 Flash Volume processing, structured output
Strategic reasoning, investor comms Claude Sonnet Depth, nuance, language precision
Draft writing, rewriting, tone Claude Sonnet Best-in-class long-form generation
Horizon analysis + sequencing Claude Sonnet Multi-step strategic logic
Side task decision trees Gemini 2.0 Flash Rapid categorization at scale
Sensitive context / personal Claude Sonnet Judgment on edge cases

Orchestration Engine: /routes/orchestrate.py

def route_to_model(task_type, payload):
    """
    Routes task to Claude or Gemini based on task type.
    Never hardcoded thresholds — all logic driven by task_type registry.
    """
    registry = load_task_registry()  # from Supabase config table
    model = registry[task_type]['model']
    
    if model == 'claude':
        return call_claude(payload, registry[task_type]['system_prompt'])
    elif model == 'gemini':
        return call_gemini(payload, registry[task_type]['system_prompt'])
    elif model == 'both':
        return call_both_and_merge(payload, registry[task_type])

The system prompt for each task type lives in Supabase, not in code. Changing a system prompt does not require a deploy.

6. SIDE TASK AGENTS

Side tasks are autonomous backend processes that run on a schedule, take actions in the world, log everything, and surface results to the operator through the dashboard. They are never destructive without explicit approval.

Architecture

Every side task follows the same contract:

1. SCAN     — read data (email, calendar, web, file)
2. CLASSIFY — use AI to categorize each item
3. DECIDE   — apply rules to generate proposed actions
4. LOG      — write every decision to side_task_log
5. GATE     — anything requiring human approval sets requires_approval=TRUE
6. REPORT   — summary surfaced on dashboard under "Side Tasks" panel
7. EXECUTE  — only approved actions are executed

Illustrative Example: Email Hygiene Agent

task_key: email_unsubscribe_sweep

What it does: The agent connects to Gmail via OAuth service account, scans all commercial/newsletter emails, and proposes unsubscribe actions based on engagement history. It never unsubscribes anything without the operator seeing it.

Decision tree

FOR EACH email thread:
  IF sender_category == 'financial'    → DO NOTHING, flag as protected
  IF sender_category == 'personal'     → DO NOTHING, flag as protected
  IF sender_category == 'legal'        → DO NOTHING, flag as protected
  IF sender_category == 'investor'     → DO NOTHING, flag as protected
  
  IF sender_category == 'commercial':
    IF never_opened == TRUE:
      → PROPOSE: unsubscribe immediately
      → requires_approval = FALSE  (auto-approve tier)
      → LOG action
    IF opened_at_least_once == TRUE:
      → PROPOSE: ask operator "Unsubscribe from [sender]?"
      → requires_approval = TRUE
      → SURFACE to dashboard for one-click decision
    IF opened_recently (< 30 days):
      → DO NOTHING, flag as 'engaged'

Categories determined by Gemini: Sender domain patterns, Subject line keywords, First-line content scan, Header List-Unsubscribe presence.

11. WHAT ANTIGRAVITY MEANS

Antigravity = net reduction in cognitive gravity per feature deployed.

Conventional productivity systems add weight: more dashboards, more apps, more tracking, more inbox management. Athena inverts this. Every layer added reduces the gravitational load on the operator. The system descends toward autonomous action on low-stakes work while elevating the operator's attention to the work only they can do.
